The Problem

AI model governance is a spreadsheet held together with prayer.

Regulators (APRA, Fed, OSFI, EU AI Office) now expect banks to prove how every model decision was made — on demand, across a multi-year retention window. The tooling to do that doesn't exist yet.

⚠️

24% of banks still on MS Excel. 40% on fragile in-house tools.

Deloitte 2025 EMEA MRM Survey (87 banks): only a third of banks have tooling that fully supports lifecycle automation or monitoring. Evidence is scattered, manual, and won't survive a regulator exam.

📋

44% of banks only sometimes validate AI/ML. 12% never do.

RMA 2024 MRM Survey: just 44% of banks with AI/ML tools said they always validate them. 12% said never. Regulators are about to close that gap — by force.

🤖

Model inventories hit 1,000+ models. Tools have not kept up.

Moody's/Risk.net 2026 survey (79 MRM leaders): all banks reported model growth. 1 in 5 commercial banks manages 1,000+ models. More than a third say current tools can't support monitoring at this scale.

The Solution

Declare models in YAML. Run one command. Get regulator-ready evidence.

RiskAttest is the glue between your MLOps stack and your GRC platform — not a replacement for either. Think dbt, but for model risk.

⌨️

Declarative YAML + CLI

Write models in YAML. mrm test, mrm docs generate, mrm replay. Same command works in dev (local) and prod (S3 + Object Lock). No notebooks required.

🔒

Hash-Chained Evidence Vault

Every EvidencePacket cryptographically chained. RFC-6962 Merkle daily roots. KMS / HSM signing. WORM-stored. Immutable from the moment of creation.

🔁

1:1 Decision Replay

Every model invocation captured as a DecisionRecord — inputs, model identity, inference params, output. Replayable byte-for-byte. OTLP-exportable to any SIEM.

🌏

Four Jurisdictions, Bundled

APRA CPS 230 · Fed SR 26-2 · EU AI Act Annex IV · OSFI E-23. Built-in, pluggable, with a 27-concept cross-standard crosswalk shipped.

🧬

GenAI & Traditional Models

14 GenAI tests (hallucination, bias, toxicity, PII, drift) plus tabular, CCR Monte Carlo, and credit risk packs. Covers sklearn, MLflow, HuggingFace, LiteLLM, Databricks.

🔌

Pluggable Architecture

Custom compliance standards via @register_standard. Custom test packs via @register_test. Entry-point discovery. 6 Architecture Decision Records documenting every design choice.

How It Works

One YAML file. One CLI. Regulator-ready output.

mrm_project.yml → mrm test → mrm evidence freeze → mrm docs generate
models:
  - name: ccr_monte_carlo
    tier: tier_1
    tests:
      - test: ccr.MCConvergence
        params: { paths: 100000, tolerance: 1e-3 }
      - test: genai.HallucinationRate
        compliance:
          cps230: ["27", "28"]
          sr26_2: ["III.A", "V.B"]
          eu_ai_act: ["Annex IV §2.b"]

# Then run:
mrm test --select tier_1
mrm evidence freeze ccr_monte_carlo
mrm replay sample --model ccr_monte_carlo --n 50
mrm docs generate --compliance standard:cps230,sr26_2

Same command in dev (local filesystem) and prod (S3 + Object Lock) — the active profile decides. No code changes across environments.
